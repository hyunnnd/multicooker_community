# 🙂 Graphene Square MultiCooker 유저 테스트

## 1. 따뜻한 환영 인사하기

안녕하세요. 오늘 Graphene Square MultiCooker 앱 유저 테스트에 참여해 주셔서 감사합니다. 이 앱은 블루투스로 멀티쿠커를 연결하고, 레시피를 찾고, 조리 단계에 맞춰 쿠커를 제어할 수 있도록 만든 서비스입니다. 식재료 사진을 활용한 AI 레시피 추천과 사용자 커뮤니티도 함께 제공합니다.

오늘은 앱을 평가하는 자리가 아니라, 처음 사용하는 분이 어떤 부분을 자연스럽게 이해하고 어떤 부분에서 멈추는지 알아보기 위한 시간입니다. 정답은 없으며, 앱이 불편하거나 기대와 다른 부분이 있다면 편하게 말씀해 주세요. 사용자의 의견이 서비스 개선에 가장 중요합니다.

테스트 중에는 가능한 한 평소처럼 앱을 사용해 주세요. 진행자는 필요한 경우에만 도움을 드리며, 화면을 누르기 전이나 기능을 찾는 동안 떠오르는 생각을 말로 설명해 주시면 좋습니다.

> **개인정보 및 촬영 동의**
>
> 테스트 기록을 위해 화면 녹화 또는 메모를 진행할 수 있습니다. 이름, 연락처, 실제 계정 비밀번호, 개인 사진 등 민감한 정보는 입력하지 않습니다. 녹화 또는 기록이 불편하면 언제든지 말씀해 주세요.

### 2. 맥락 질문하기

멀티쿠커, 에어프라이어, 전기밥솥처럼 조리 기기를 사용해 본 경험을 중심으로 가볍게 대화를 시작합니다.

- 평소 집에서 어떤 조리 기기를 가장 자주 사용하시나요?
- 레시피를 찾을 때 주로 어떤 앱이나 서비스를 사용하시나요?
- 조리할 때 시간이나 온도를 따로 맞추는 과정에서 불편했던 점이 있나요?
- 냉장고 재료로 만들 수 있는 메뉴를 추천받는 기능을 사용해 본 적이 있나요?
- 조리 후기나 레시피를 다른 사람과 공유한 경험이 있나요?

이제 위와 같은 불편을 줄이기 위해 Graphene Square MultiCooker 앱을 만들었습니다. 앱을 사용하면서 앞에서 말씀하신 경험과 비교해 느낀 점을 편하게 말씀해 주세요.

## 3. User Test

> **모든 의사 결정에서 사용자의 생각이 최우선입니다.**

사용자가 서비스를 사용하며 무엇을 기대하는지, 어느 화면에서 이해가 어려운지 관찰합니다. 사용자의 행동을 보면서도 기능을 설명하거나 정답을 알려주지 않습니다. 사용자의 관점에서 처음부터 자연스럽게 사용할 수 있는지를 확인하는 것이 목적입니다.

### 1. 역할을 정해주세요

- **UT 진행자**: 테스트 흐름을 안내하고 중립적인 질문을 하는 사람
- **행동 관찰자**: 사용자의 표정, 머뭇거림, 화면 전환과 발화를 기록하는 사람
- **기록 담당자**: 과제별 성공 여부, 소요 시간, 인용할 만한 발화를 기록하는 사람
- **기기 담당자**: 테스트용 멀티쿠커와 블루투스 상태를 준비하고, 안전상 필요한 경우에만 개입하는 사람
- **UT 테스터(사용자)**: 처음 사용하는 사람의 입장에서 자유롭게 앱을 탐색하는 사람

### 2. 유저 테스트 5단계

> **모든 테스트는 한 사람당 20-25분을 넘기지 않는 것을 권장드립니다.**

#### 따뜻한 환영 인사하기

사용자가 편안하게 느끼도록 테스트 목적, 녹화 여부, 진행 방식을 먼저 안내합니다. 앱이나 사용자를 평가하는 것이 아니라 화면과 흐름을 개선하기 위한 자리임을 다시 알려드립니다.

#### 맥락 질문하기

평소 요리 습관, 조리 기기 사용 경험, 레시피 탐색 방식에 대한 간단한 질문으로 시작합니다. 본격적인 과제로 넘어가기 전 사용자의 기준과 기대를 이해합니다.

#### 프로토타입 소개하기

"지금부터 Graphene Square MultiCooker 앱을 처음 사용한다고 생각하고 이용해 주세요. 일부 기능은 테스트용 계정이나 기기로 동작하며, 실제 조리를 시작하기 전에는 진행자가 안전을 확인합니다."라고 안내합니다.

사용자가 과제를 해결하는 동안 화면의 기능을 먼저 설명하지 않습니다. "여기를 눌러 보세요" 대신 "어떤 기능일 것 같나요?", "다음에는 무엇을 해보고 싶으세요?"처럼 생각을 듣는 질문을 사용합니다.

#### 관찰을 통한 인사이트 얻기

사용자가 스스로 목적지를 찾는 과정을 지켜봅니다. 길을 잃거나 멈춘 경우에도 바로 해결 방법을 알려주지 않고, 어떤 문구나 아이콘이 헷갈렸는지 질문합니다. 과제 종료 후에는 그 행동을 다시 짚어 이유를 확인합니다.

#### 맺음말

마지막으로 전체 인상, 가장 좋았던 점, 가장 불편했던 점, 앞으로 추가되었으면 하는 기능을 듣습니다. 참여에 감사드리고 필요 시 보상 및 후속 연락 방법을 안내합니다.

## 4. 유저 테스트 진행 스크립트

### [진행자] 1. 따뜻한 환영 인사 및 맥락 질문

안녕하세요. Graphene Square MultiCooker 유저 테스트에 참여해 주셔서 감사합니다. 오늘은 멀티쿠커를 연결하고 레시피를 골라 조리까지 이어지는 흐름이 처음 사용자에게 자연스러운지 확인하려고 합니다. 테스트 중 불편한 점이나 이해되지 않는 점은 솔직하게 말씀해 주세요. 기능을 잘못 사용하셔도 전혀 괜찮습니다.

먼저 평소 요리할 때 레시피를 어떻게 찾으시는지, 그리고 조리 기기를 사용하면서 어려웠던 점이 있었는지 들려주실 수 있을까요?

### [진행자] 2. 첫 진입 및 튜토리얼

**과제**: 로그인 후 처음 보이는 홈 화면에서 앱이 어떤 서비스인지 파악해 주세요. 화면에 튜토리얼이 나타나면 안내를 따라가 보시고, 끝난 뒤 기억나는 기능을 말씀해 주세요.

**관찰 포인트**

- 홈 화면의 연결 상태 카드와 오늘의 추천 레시피를 구분하는지
- 스포트라이트 튜토리얼의 순서와 문구를 이해하는지
- 튜토리얼을 건너뛰거나 다시 보는 위치를 찾을 수 있는지
- 한돌이 캐릭터가 안내를 방해하는지 또는 도움이 되는지

**후속 질문**

- 처음 화면에서 가장 먼저 눈에 들어온 것은 무엇이었나요?
- 튜토리얼이 안내한 기능 중 기억나는 것이 있나요?
- 한돌이는 어떤 역할을 한다고 느끼셨나요?

### [진행자] 3. 쿠커 연결과 연결 오류 안내

**과제**: "쿠커를 연결해서 레시피 조리를 준비해 보세요." 홈 화면의 연결 상태 카드를 눌러 기기 관리 화면으로 이동하고, 주변의 테스트용 Graphene Cooker를 찾아 연결해 주세요.

**연결 성공 시 관찰 포인트**

- 홈 화면의 연결 상태가 바뀐 것을 알아차리는지
- 기기 이름과 연결 완료 상태를 이해하는지
- 연결 후 한돌이의 상태 변화가 자연스럽게 느껴지는지

**재연결 또는 블루투스 꺼짐 상태 시 과제**: "현재 연결이 끊겼거나 블루투스가 꺼진 상태입니다. 화면에서 무슨 상황인지 파악하고, 다시 연결할 수 있는 방법을 찾아 주세요."

**관찰 포인트**

- 주황색 재연결 상태와 빨간색 블루투스 꺼짐 상태의 차이를 이해하는지
- 자동 재연결 횟수(예: 1/3)와 안내 문구를 알아차리는지
- 한돌이 말풍선 또는 상태 카드의 이동 단서를 눌러 기기 관리 화면으로 갈 수 있는지
- 연결 실패 시 다음 행동을 스스로 결정할 수 있는지

**후속 질문**

- 지금 연결 상태를 한 문장으로 설명해 주실 수 있나요?
- 다시 연결하려면 무엇을 해야 한다고 생각하셨나요?
- 자동 재연결을 기다리는 흐름이 충분히 이해되었나요?

> **진행 유의**: 실제 기기 연결은 테스트용 쿠커에서만 진행합니다. 기기가 없거나 연결이 어려우면 재연결/어댑터 꺼짐 상태 화면을 보여주고, 사용자의 이해 여부만 확인합니다.

### [진행자] 4. 레시피 탐색과 조리 시작

**과제**: "오늘 저녁에 만들 메뉴를 찾는다고 생각하고 레시피를 하나 골라 조리를 시작해 주세요." 홈의 오늘의 추천 또는 하단 레시피 탭에서 원하는 레시피를 찾아 상세 화면으로 이동합니다.

**관찰 포인트**

- 레시피 목록, 검색, 카테고리에서 탐색 방법을 선택하는지
- 조리 시간, 조리 방식, 공식 레시피 표시를 이해하는지
- 레시피 상세에서 재료와 단계 정보를 확인하는지
- `조리하기` 흐름을 찾고, 쿠커 미연결 시 안내를 이해하는지

**연결된 기기에서의 추가 과제**: "예열과 실제 조리 시간이 어떻게 구분되는지 확인한 뒤, 조리를 시작할 수 있을지 판단해 주세요."

**관찰 포인트**

- 예열 완료 전 실제 조리 시간이 시작되지 않는다는 점을 이해하는지
- 조리 단계와 쿠커 설정이 연결되어 있다는 점을 파악하는지
- 조리 중 한돌이의 요리 동작이 현재 상태와 어울린다고 느끼는지

**후속 질문**

- 이 레시피를 선택한 이유는 무엇인가요?
- 조리 시작 전 가장 확인하고 싶었던 정보는 무엇인가요?
- 연결되지 않은 상태에서 조리하려 할 때 다음 행동이 명확했나요?

### [진행자] 5. AI 식재료 인식과 추천

**과제**: "냉장고에 남은 식재료로 만들 수 있는 메뉴를 찾고 싶습니다. AI 추천을 이용해 보세요." 하단 AI 탭으로 이동해 촬영 또는 앨범에서 테스트 이미지를 선택하고, 분석 결과와 추천 레시피를 확인합니다.

**관찰 포인트**

- 촬영과 앨범 선택의 차이를 이해하는지
- 식재료가 잘 보이도록 촬영하라는 가이드가 충분한지
- 분석 후 추천 결과가 무엇을 의미하는지 이해하는지
- 추천 레시피에서 다음 행동(상세 보기, 조리 시작)을 찾는지

**후속 질문**

- AI가 추천한 결과를 신뢰할 수 있을 것 같나요? 그 이유는 무엇인가요?
- 결과 화면에서 더 보고 싶은 정보가 있나요?
- 사진을 올리는 과정에서 걱정되거나 불편한 점이 있었나요?

### [진행자] 6. 커뮤니티와 개인 메뉴 탐색

**과제**: "다른 사용자의 조리 후기를 읽고, 내가 저장한 레시피 또는 조리 이력을 찾아보세요." 하단 커뮤니티 탭과 설정/마이 메뉴를 자유롭게 탐색합니다.

**관찰 포인트**

- 커뮤니티 글, 후기, 댓글의 구분을 이해하는지
- 저장한 레시피와 조리 이력의 위치를 쉽게 찾는지
- 조리 이력에서 같은 설정으로 다시 조리하는 행동을 예상할 수 있는지
- 앱바의 뒤로가기와 화면 제목이 현재 위치를 충분히 알려주는지

**후속 질문**

- 가장 유용하다고 느낀 개인 메뉴는 무엇인가요?
- 커뮤니티에서 보고 싶은 정보 또는 기능이 있나요?
- 홈, 레시피, AI, 커뮤니티, 설정 탭의 역할이 구분되었나요?

### [진행자] 7. 맺음말 및 질문

- 오늘 사용한 기능 중 가장 좋았던 점은 무엇인가요?
- 가장 헷갈리거나 불편했던 순간은 언제였나요?
- 실제로 이 앱을 사용한다면 가장 자주 쓸 기능은 무엇인가요?
- 조리 전, 조리 중, 조리 후에 추가되었으면 하는 기능이 있나요?
- 한돌이 캐릭터의 크기, 위치, 움직임, 말풍선 안내에 대해 개선할 점이 있나요?
- 이 앱을 지인에게 추천할 의향이 있나요? 이유를 말씀해 주세요.

오늘 테스트에 참여해 주셔서 감사합니다. 말씀해 주신 의견은 Graphene Square MultiCooker를 더 쉽고 안전하게 사용할 수 있도록 개선하는 데 활용하겠습니다.

## 6. 유저 테스트 주의사항

### 친절한 진행자가 되어주세요

인터뷰 전 과정에서 사용자가 부담을 느끼지 않도록 편안하게 대합니다. 원하는 답을 얻기 위해 유도하지 않고, 사용자가 스스로 생각을 표현할 수 있게 기다립니다.

### 네/아니오 형식의 질문은 피하세요

"좋았나요?"보다 "어떤 점이 좋거나 아쉬웠나요?"처럼 열린 질문을 사용합니다. 다지선다형으로 답을 제한하지 않습니다.

### 필요한 질문은 중간에 끊으세요

사용자가 말하거나 행동하는 중에는 충분히 관찰한 뒤, 자연스럽게 멈춘 시점에 질문합니다. 진행자가 설명하는 시간보다 사용자의 이야기를 듣는 시간을 더 길게 확보합니다.

### 호기심에 대한 사과 방식

사용자의 반응과 사고에 진심으로 공감합니다. "왜 그렇게 하셨어요?"보다 "그렇게 생각하신 이유를 조금 더 들려주실 수 있을까요?"라고 묻습니다.

### 개인정보 동의를 받으세요

- 화면 녹화, 사진, 음성 녹취가 필요하면 시작 전 별도 동의를 받습니다.
- 실제 계정 비밀번호, 개인 연락처, 개인 사진을 입력하거나 기록하지 않습니다.
- 블루투스 권한, 카메라·앨범 권한은 테스트 목적과 범위를 설명한 뒤 받습니다.
- 불편하거나 원하지 않는 기능은 사용자가 거부할 수 있으며, 강요하지 않습니다.
- 실제 쿠커 조리는 화상·화재 위험이 없도록 담당자가 현장을 관리합니다.

## 7. 피드백 공유

모든 인터뷰가 끝나면 팀원과 기록을 공유하고, 사용자 발화를 기준으로 개선 우선순위를 정합니다.

- 긍정적 피드백과 불편 피드백을 분리해 정리합니다.
- 성공률이 낮거나 멈춤 시간이 긴 과제를 우선 개선합니다.
- 연결 상태, 권한 요청, 조리 시작처럼 안전·신뢰와 관계된 문제를 최우선으로 검토합니다.
- 다음 테스트에서 검증할 가설과 수정안을 명확히 기록합니다.
