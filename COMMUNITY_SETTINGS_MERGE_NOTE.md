# Community / Settings merge note

이 프로젝트는 `GrapheneSquare_cooker_app-master (10).zip`을 기준으로 화면과 기존 조리·레시피 흐름을 유지하고, 이전 개발본의 커뮤니티·설정·알림·로컬 API 기능을 통합한 버전입니다.

## 통합된 주요 기능

- 커뮤니티 게시글, 댓글, 답글, 좋아요, 신고, 차단, 알림
- 게시글 임시 저장 및 복원
- 작성자 프로필, 공개 레시피 조회, 이미지 확대, 수정 이력
- 인기 배지를 카테고리 배지 왼쪽에 같은 줄로 표시
- 댓글·답글에서 게시글 작성자에게만 `작성자` 표시
- 설정 세분화: 댓글, 답글, 좋아요, 공지, 조리 알림
- 차단 사용자 관리, 자동 재연결, 슬라임 표시, 언어, 튜토리얼 설정
- 프로필 이미지 및 닉네임 동기화
- 로컬 알림과 Firebase 원격 알림 연결
- 공식 레시피와 공개 사용자 레시피를 `/recipes/feed`로 통합 조회
- 레시피 업로드를 `/users/me/recipes`에 연결하고 신규 업로드는 현재 공유 UI에 맞춰 공개로 저장

## API 구분

- 회사 인증 API 기본값: `http://3.36.14.110:8000`
- 커뮤니티·레시피·설정·기기·AI 로컬 API 기본값: `http://192.1.0.28:8001`

실행 환경에 맞게 다음 값을 지정할 수 있습니다.

```bash
flutter run \
  --dart-define=AUTH_API_BASE_URL=http://인증서버주소 \
  --dart-define=API_BASE_URL=http://로컬API주소
```

## 서버 적용

통합본의 `main.py`와 `requirements.txt`를 사용해야 추가된 커뮤니티·설정·프로필·공개 레시피 API가 동작합니다. 기존 `multicooker.db`, `.env`, `local_uploads`를 유지한 상태에서 서버 코드를 교체하면 시작 시 마이그레이션이 적용됩니다. 실제 운영 전에는 DB를 백업하십시오.

## 앱 의존성

통합 후 다음 명령을 실행하십시오.

```bash
flutter pub get
```

Firebase 설정 파일이 없는 개발 환경에서는 앱 실행 중 폴링 기반 로컬 알림으로 동작하며, Firebase 설정이 완료된 빌드에서는 원격 푸시 등록을 사용합니다.
