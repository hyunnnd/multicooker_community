# 커뮤니티 관리자 기능

## 관리자 지정

프로젝트 루트 `.env`에 회사 로그인 이메일을 등록합니다.

```env
ADMIN_EMAILS=admin@example.com
POPULAR_SCORE_THRESHOLD=3
```

여러 명은 쉼표로 구분합니다.

```env
ADMIN_EMAILS=admin1@example.com,admin2@example.com
```

앱에서 해당 이메일로 로그인한 뒤 커뮤니티 상단의 방패 아이콘을 누르면 관리자 화면이 열립니다.

## 관리자 화면

- 공지 관리: 공지 작성, 수정, 삭제, 중요 공지 상단 고정
- 신고 관리: 전체/미처리/처리완료/반려 건수, 신고 내용 확인, 반려, 처리 완료, 콘텐츠 삭제
- 인기 관리: 게시글 좋아요 표시 수, 관리자 인기도 보정값, 강제 인기 여부 설정

## 인기글 계산

인기 점수는 다음 기준입니다.

```text
최근 N일 게시글 좋아요 수 + (최근 N일 댓글·답글 수 × 2) + 관리자 보정값
```

기본적으로 3일을 사용하며 활동이 없으면 6일, 9일, 12일 순으로 범위를 넓힙니다. `force_popular`이 켜진 게시글은 인기 목록 상단에 우선 표시됩니다.

이전 버전처럼 누적 활동 수를 3일 활동으로 계속 보관하지 않고 `created_at`을 기준으로 실제 기간을 계산합니다.

## 서버 적용

`main.py`를 교체하고 서버를 재시작합니다. 기존 `multicooker.db`는 삭제하지 않아도 새 열이 자동 추가됩니다.
