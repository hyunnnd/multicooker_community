# 알림·설정·닉네임 동기화 수정 내역

## 1. 자기 행동 알림 제외

다음 행동은 본인에게 알림을 만들지 않도록 서버에서 차단했습니다.

- 자신의 게시글에 직접 댓글 작성
- 자신의 댓글에 직접 답글 작성
- 자신의 공개 레시피에 직접 댓글 작성
- 자신의 공개 레시피에 직접 후기 작성

기존 데이터에 남아 있는 자기 알림도 알림 조회 시 사용자 ID를 비교하여 표시하지 않습니다.

## 2. 사용자 공유 레시피 제작자 이름 동기화

공개 사용자 레시피의 제작자명은 저장 당시 문자열만 사용하지 않고 현재 소유자 계정의 닉네임을 기준으로 응답합니다.
닉네임 변경 시 다음 데이터도 함께 갱신합니다.

- 공개·비공개 개인 레시피의 제작자명
- 다른 사용자가 저장한 해당 레시피의 스냅샷 제작자명
- 게시글, 댓글, 답글, 레시피 댓글, 후기 작성자명
- 기존 알림의 발신자 표시명

앱에서 프로필 저장 직후 커뮤니티와 레시피 목록을 다시 조회하여 화면에도 즉시 반영합니다.

## 3. 설정의 계정 영역 정리

상시 표시할 필요가 없었던 `로그인됨`, `인증 상태 확인`, `이메일 인증` 항목은 제거했습니다.
현재는 다음만 표시합니다.

- 로그인 사용자의 닉네임과 이메일
- 암호 변경
- 로그아웃
- 비로그인 상태의 로그인 버튼

이메일 인증은 회원가입 및 비밀번호 재설정 흐름에서만 사용합니다.

## 4. 앱 동작 설정 연결

다음 설정을 FastAPI 사용자 설정에 저장하고 앱 재실행 후에도 다시 불러옵니다.

- 레시피·커뮤니티 알림
- 조리 알림
- 쿠커 자동 재연결
- 언어

## 5. 휴대전화 로컬 알림

### 레시피·커뮤니티

- 같은 알림 ID를 갱신하여 알림 창에는 최신 알림 한 건만 유지합니다.
- 읽지 않은 알림 수를 숫자와 보조 문구로 표시합니다.
- 새 알림이 아니라 읽지 않은 개수만 바뀌는 경우 소리와 진동을 다시 발생시키지 않습니다.
- 알림을 누르면 해당 게시글 또는 레시피 상세로 이동합니다.

### 조리

- 예열 완료 알림
- 조리 완료 알림

### 자동 재연결

- 연결 유실 후 자동 재연결을 시도하는 동안 재연결 알림을 표시합니다.
- 재연결 성공, 스캔 종료 또는 자동 재연결 해제 시 알림을 제거합니다.
- Android에서는 진행 중 알림으로 유지합니다.

## 6. 현재 동작 범위

현재 구현은 앱 내부 상태와 로컬 알림을 연결한 방식입니다.
커뮤니티 알림은 앱이 실행 중이거나 다시 활성화된 상태에서 서버를 주기적으로 확인합니다.
앱이 완전히 종료된 상태에서도 즉시 수신하려면 향후 FCM/APNs, 기기 토큰 저장, 서버 푸시 전송 구성이 추가로 필요합니다.

## 7. 플랫폼 설정

- `flutter_local_notifications` 의존성 추가
- Android 알림 권한 추가
- Android `compileSdk 36`, `minSdk 24`, core library desugaring 적용
- iOS 알림 응답 delegate 연결
